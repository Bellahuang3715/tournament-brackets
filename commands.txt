How to check “what will publish” + vulnerabilities before publish
A) Confirm tarball contents (must-do)
npm pack --dry-run

You should see basically dist/ + README + LICENSE + package.json.

B) Check consumer-facing vulnerabilities (runtime deps only)
npm audit --omit=dev

This is the important one for your users.

C) Check everything (including your toolchain)
npm audit

But it's okay if this one has vulnerabilities. As long as B comes out clean

"npm version patch" to update patch version
"npm publish" to republish to npm
